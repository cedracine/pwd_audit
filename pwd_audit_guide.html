🗺 Ce qu'on va faire

Tu as des mots de passe éparpillés dans Chrome, Firefox, Opera, Safari, Vivaldi… certains datent de 20 ans. On va tout fusionner, trier, évaluer la force de chaque pwd, détecter les sites morts — et finalement importer le tout proprement dans Proton Pass.

🧰 Outils nécessaires

Python 3.8+ — gratuit, à installer si absent

zxcvbn — librairie de scoring, 1 commande à taper

pwd_audit_gui.py — le script fourni avec ce guide

💡Tout tourne en local sur ta machine — aucun mot de passe n'est envoyé sur internet.
Étape 1 — Installer Python & la librairie

🪟 Windows 10/11

1. Va sur python.org/downloads 2. Télécharge Python 3.12 3. Lance l'installeur — coche Add Python to PATH 4. Ouvre PowerShell et tape :

python --version # doit afficher Python 3.x.x pip install zxcvbn

🍎 macOS

Python 3 est déjà installé sur macOS récent. Ouvre le Terminal :

python3 --version # si absent : brew install python3 pip3 install zxcvbn

🐧 Raspberry Pi / Linux

Python 3 est inclus dans TwisterOS. Ouvre un terminal :

python3 --version pip3 install zxcvbn \ --break-system-packages

✅ Vérifier que tout marche

Dans le terminal, copie-colle :

python3 -c "import zxcvbn; \ print('zxcvbn OK')"

Tu dois voir : zxcvbn OK — Si erreur → retente pip install

⚠️Sur Windows, si python n'est pas reconnu après install, ferme et rouvre PowerShell. Ou essaie py à la place de python.
Étape 2 — Exporter les CSV depuis chaque navigateur
🔴Les CSV exportés contiennent tous tes mots de passe en clair. Mets-les dans un dossier temporaire dédié, et supprime-les après l'import.
Chrome / Edge / Brave
1. chrome://password-manager/settings 2. Mots de passe enregistrés 3. ⋮ → Exporter
name · url · username · password
Vivaldi, Opera, Chromium : même procédure via chrome://settings/passwords
Firefox
1. about:logins 2. ⋯ → Exporter les identifiants 3. Confirme mot de passe système
url · username · password · guid · timeCreated · timePasswordChanged
Timestamps Firefox en millisecondes Unix — convertis automatiquement.
Safari
1. Safari → Réglages 2. Mots de passe 3. Touch ID 4. ⋯ → Exporter
Title · URL · Username · Password · OTPAuth · Notes
⚠️ Désactive iCloud Drive avant l'export.
Vivaldi
1. Menu Vivaldi → Fichier → Exporter mots de passe — OU — vivaldi://settings/privacy
name · url · username · password · note
Même format que Chrome — détecté automatiquement.
Opera
1. Paramètres (Alt+P) 2. Avancé → Sécurité 3. Exporter les mots de passe
name · url · username · password
Identique au format Chrome.
Proton Pass
1. Extension Proton Pass 2. ☰ → Paramètres 3. Export → CSV
name · url · email · username · password · note · totp · vault
Utile pour consolider un export Proton Pass existant.
💡Mets tous tes CSV dans un même dossier, ex: C:\pwd_audit\ ou ~/pwd_audit/
Étape 3 — Lancer pwd_audit_gui.py

Copie le fichier script

Place pwd_audit_gui.py dans le même dossier que tes CSV, ou n'importe où sur ton disque.

WindowsmacOSLinuxRaspi

Ouvre un terminal dans ce dossier

Windows : Shift+clic droit → "Ouvrir PowerShell ici" — macOS : clic droit → "Nouveau terminal ici" — Linux/Raspi : clic droit → "Ouvrir dans un terminal"

Lance le script

# Windows python pwd_audit_gui.py # macOS / Linux / Raspi python3 pwd_audit_gui.py

Une fenêtre s'ouvre — c'est l'interface graphique.

Onglet 📂 Fichiers CSV

Clique "Ajouter CSV(s)" et sélectionne tous tes fichiers exportés.

Onglet 🔍 Audit → Lancer

Choisis le dossier de sortie. Laisse cochée "Vérifier les domaines morts". Clique "Lancer l'Audit". Durée : 2–5 min.

📊 Fichiers générés

passwords_clean.csv — toutes les entrées triées, score de force inclus. C'est ce fichier qu'on importe dans Proton Pass.

sites_dead.csv — domaines qui n'ont pas répondu. Comptes probablement obsolètes.

sites_to_review.csv — domaines ambigus. À vérifier à la main.

Étape 4 — Importer dans Proton Pass
🔑 Format attendu par Proton Pass (Generic CSV)

Proton Pass accepte un CSV avec exactement ces colonnes :

Colonne Obligatoire Description Exemple
name✅ RequisNom/label de l'entréegithub.com
url✅ RequisURL complète du sitehttps://github.com
username✅ RequisNom d'utilisateurceddev
password✅ RequisMot de passeGh!x9Lm2#qR7
email○ OptionnelEmail associéced@pm.me
note○ OptionnelNotes libres—
totp○ OptionnelSecret 2FA (otpauth://)otpauth://…
vault○ OptionnelVault de destinationPersonal

Ouvre Proton Pass

Clique sur l'extension Proton Pass ou va sur pass.proton.me

Paramètres → Import

☰ → Paramètres → section Import

Sélectionne "Generic CSV"

Choisis Generic CSV dans la liste. Clique Choisir un fichier → sélectionne passwords_clean.csv.

Choisis le vault de destination

Sélectionne un vault existant ou crée-en un nouveau. Tu pourras réorganiser après.

Confirme et vérifie

Clique Importer. Vérifie que le nombre d'entrées correspond. Ensuite : supprime les CSV de ton disque.

🔴Supprime tous les CSV après import. Ce sont des fichiers en clair avec tous tes mots de passe.
Étape 5 — Organiser les Vaults dans Proton Pass

Une fois l'import fait, répartis tes comptes dans des vaults thématiques :

🔴
Critical
Email principal, banque, Proton, hébergement, domaines, GitHub
🟡
Work / Dev
APIs, serveurs, DNS, registrars, outils dev, hosting
🟢
Personal
Réseaux sociaux, shopping, streaming, services courants
🔵
Old / Unsure
Comptes douteux, sites peut-être morts, à vérifier progressivement
⚫
Shared
Accès partagés famille / équipe
💡Labels utiles dans Proton Pass : 2FA-enabled, pwd-à-changer, pwd-réutilisé, hibp-compromis
📅 Stratégie de changement de mots de passe

Ne change pas tout d'un coup — tu vas abandonner. Approche progressive :

🔴 Semaine 1 : Critical seulement (email, banque, Proton lui-même)

🟡 Semaine 2–3 : Comptes avec score 0–1 sur des sites actifs

🔄 En continu : À chaque connexion → laisser Proton Pass générer un nouveau pwd fort (20+ chars)
